# m-cns-guide

## 项目简介

m-cns-guide 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或具体网站，仅作为一系列独立 HTML 文档的集中存放与版本管理之用。

这些页面内容涉及信息指引、文档集合或工具型页面，各自独立运行，不相互依赖。仓库的核心目标是提供一个干净、可追溯的 HTML 页面存档环境。

## 目录说明

```
m-cns-guide/
├── README.md          # 本文件
├── index.html         # 仓库首页 / 页面索引
├── pages/             # 存放各独立 HTML 页面
│   ├── example-1.html
│   ├── example-2.html
│   └── ...
└── assets/            # 公共资源（CSS、JS、图片等）
    ├── style.css
    └── script.js
```

- **pages/**：所有独立 HTML 页面存放于此，每个页面为一个独立的文档。
- **assets/**：存放页面间共享的样式、脚本、图片等静态资源。

## 页面归档说明

- 每个 HTML 页面均为独立文件，可直接在浏览器中打开运行。
- 页面之间无硬性链接依赖，但可自愿通过 `index.html` 进行索引。
- 页面内容由仓库维护者按需归档，不对外提供托管或代理服务。
- 归档内容遵循通用信息归档原则，不针对任何特定主体。

## 维护说明

- 仓库采用常规 Git 版本管理，每次更新应附有清晰的提交信息。
- 新增页面请放入 `pages/` 目录，并确保文件命名简洁、无歧义。
- 若页面引用 `assets/` 中的资源，请使用相对路径。
- 不建议在页面中嵌入外部不可控资源，以保持归档的长期可用性。
- 如需删除或修改归档页面，请通过 Pull Request 或直接提交，并说明理由。

## 许可

本仓库内容仅供归档与参考之用，不构成任何形式的建议或承诺。具体使用请遵守相关法律法规及平台政策。
